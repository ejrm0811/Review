[인트로 화면 전환 구현 순서]

1. 안드로이드 APP 설계 문서
    
    * config  (환경셜정에 관한 영역)
    a. 지원 영역 :  android sdk version 19 ~ 27 까지 지원하는 APP 을 작성하겟다
    b. 타겟 sdk는 version 27로 잡겟다
    c. 주변기기(프린트)의 스펙을 고려하여 환경 설정을 진행한다.

    

    * 개발 영역에 관한 설게(UX USER EXPERIENCE)
    1. APP  첫기동은 런처 이미지 클릭으로 시작한다(OS 단 클릭 이벤트)
    2. 런처 이미지 클릭시 인트로(IntAct.java onCreate() ) 화면을 약 3초간 보여준다() 

    
    * 디자인 설계(500줄)
    1. IntAct.java >> 백그라운드 이미지 하나면 완료
    2. MainAct.java >> 헤더이미지와 버튼 5개로 구성 탭은 3곳으로 구성 

    -> 시나리오

    1. 어플을 실행시킵니다.

    2. 어플의 초기화면의 인트로가 3초동안 출력되고 메인 화면으로 이동합니다.

    3. 메인화면에는 3개의 버튼이 있는데 "회사소개", "상품 설명", "주문하기" 입니다.

    4. 주문하기 버튼을 클릭하면 주문화면으로 이동됩니다.
        
        상품의 가격은 전역변수(Price_1)로 정의되어 모든 곳에서 사용할 수 있도록 사전에 정의합니다.
        이미지 파일은 drawable에 미리 위치한다.

    5. 주문화면에 이미지 뷰를 출력하고 수량버튼(EditText)을 클릭하면 수량을 입력할 수 있는 자판이 화면에 출력됩니다.
        ※ 잘못 클릭했을 경우 뒤로가기 버튼을 누르면 다시 주문화면으로 돌아갑니다.
    
    6. 입력된 수량은 ? 무슨 변수(Menu_1,Menu_2)에 순차적으로 저장됩니다.

    7. 주문완료 버튼을 클릭하면 선택한 변수들을 나열한 화면으로 이동합니다.

    8. 저장된 Menu_1 변수와 미리 정의해 놓은 전역변수 Price_1 변수를 곱하는 클래스를 정의하고 화면에 출력합니다.
